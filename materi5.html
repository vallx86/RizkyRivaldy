DNS

Apa itu DNS Server ?

DNS artinya adalah sebuah layanan yang di install pada sebuah server untuk menyimpan informasi tentang nama host atau nama domain dalam bentuk database atau basis data tersebar (distributed database) pada jaringan komputer.

Dalam layanan DNS disediakan alamat IP address bagi setiap host atau email elektronik untuk setiap domain, dengan adanya DNS sebuah pusat layanan jaringan atau lebih dikenal dengan istilah komputer server, bukan hanya memiliki alamat IP address, namun memiliki sebuah nama panggilan yang dinamakan domain.

Jadi dengan nama domain sobat mudah untuk mengingat dan mengakses sebuah server, contoh paling sederhana adalah sebuah website di internet, semua domain atau alamat Uniform Resource Locator (URL) itu adalah awalnya sebuah alamat IP address berupa nomor (biasa muncul di taskbar sudut kiri bawah), dengan adanya domain atau penamaan dalam bentuk huruf, maka sobat mudah untuk mengingat dan membukanya menggunakan browser.

Sejarah DNS

Pertama berdirinya ARPAnet bahwa DNS dulunya belum ada dan komputer hanya menggunakan host.txt yang memertakan nama ke seluruh komputer yang terhubung dengan jaringan. Sampai saat ini masih terdapat file host.txt pada beberapa sistem operasi. Dengan sistem tersebut sangat kesulitan apabila alamat pada suatu host berubah, sehingga setiap host yang akan terhubung harus melakukan update terhadap file host.

Seiring waktu berjalan jaringan komputer semakin berkembang. Paul Mockapetris menemukan DNS pada tahun 1983, saat itu muncul dengan spesifikasi Reques For Comment (RFC) seri 882 dan 883, disusul kemudian tahun 1987 dengan RFC 1034 dan RFC 1035 hingga saat ini.

Macam-macam DNS

Pengalamatan sebuah domain di atur sedemikian rupa oleh ARPAnet agar tidak terjadi adanya double alamat, sehingga sebuah jaringan private atau local yang hendak terhubung jaringan, harus memiliki sebuah IP public yang biasa diberikan oleh Internet Service Provider (ISP). Bentuk hostname yang dipakai di internet sangat mirip dengan alamat IP address local. Berikut adalah beberapa domain yang harus dipahami.


1. Generic Top Level Domain (GTLD)

Dimiliki oleh Negara Amerika, namun sering digunakan dan terlihat seperti domain umum dalam alamat internet. Sebagian dari pengguna internet banyak memilih domain level tinggi ini. Semua orang dapat mempunyai domain ini tanpa harus menggunakan persyaratan yang rumit, kecuali beberapa domain seperti edu, mil dan travel.

Fungsi DNS Server

DNS server memiliki fungsi sebagai database server yang menyimpan alamat IP address untuk memberikan nama pada hostname. Ketika sobat mengetikan nama domain atau alamat URL seperti google.com, secara tidak langsung sobat mengetikan sebuah IP address dengan nomor 8.8.8.8. Dari contoh tersebut maka terdapat 2 fungsi dari DNS server :

1. DNS server berfungsi untuk melakukan modulator atau menterjemahkan sebuah alamat IP address, mejadi sebuah nama domain.
2. DNS digunakan untuk mengetahui data secara lengkap mengenai serbuah hostname yang ada di internet baik alamat IP address, alamat lokasi server dan waktu hostname digunakan.

Kelebihan dan Kekurangan DNS Server

Dari fungsi yang telah diketahui bersama, terdapat kelebihan dan kekurangan yang harus diketahui :

Kelebihan DNS server

1. Mempermudah, artinya DNS sangat mempermudah sobat untuk membuka alamat hostname, cukup mengingat sebuat nama dengan tidak mengingat berapa alamat IP address hostname tersebut.
2. Konsisten, artinya alamat IP address pada sebuah komputer dapat berubah, namun dengan adanya DNS hostname tidak perlu melakukan update.
3. Sederhana, artinya mudah sekali untuk dikonfigurasi

Kelemahan DNS Server

Sebagai user tidak dapat menggunakan nama lebih dari satu untuk melakukan pencarian domain, baik di jaringan local maupun di jaringan global (internet)

Aplikasi DNS Server

Beberapa aplikasi yang dapat dipergunakan sebagai DNS server pada beberapa sistem operasi diantaranya :

    1. Berkeley Internet Name Domain (BIND) : sebuah aplikasi DNS server yang banyak digunakan di jaringan global, menggunakan sistem operasi unix atau linux.
    2. Mara DNS : Perangkat lunak DNS yang gratis dibuat oleh Sam Trenholme, namun aplikasi ini tidak banyak yang menggunakan
    3. DjbDNS : Kumpulan aplikasi DNS termasuk tinydns yang digunakan pada awal tahun 2004 dirancang oleh Daniel J Bernstein.
    4. Name Server Daemon (NSD) : NSD server merupakan perangkat lunak bebas otoritatif yang diciptakan oleh NLNet Labs.
    5. Unbound : Diciptakan juga oleh NLNet Labs dan sempat ditawarkan ke Linux yang terintegrasi dengan protocol routing quagga pada peralatan Dell/TCPWave.
    6. Power DNS : Perangkat lunak server gratis dengan beberapa tempat penyimpanan data dan fitur load balancing.
    7. Microsoft DNS : Diciptakan oleh Microsoft dan di pasang pada sistem operasi Windows versi server seperti Windows 2000 server, 2003 server, 2008 server, 2012 server dan 2016 server.

Cara Kerja DNS Server

Sebelum masuk ke pembehasan cara kerja DNS, sebelumnya sobat pahami terlebih dahulu bahwa DNS dikelola dengan 3 komponen yaitu :

    1. DNS resolver : komputer client yang disebut pengguna, melakukan permintaan DNS dari sebuah sistem operasi
    2. Recursive DNS Server : pihak yang melakukan pencarian melalui DNS dan berdasarkan permintaan resolver, selanjutnya diberikan jawaban pada resolver tersebut.
    3. Authoritative DNS Server : pihak yang memberikan respon setelah recursive akan melakukan pencarian dan selanjutnya memberikan respon atau jawaban yang didelegasikan ke DNS server lainnya.

      Pada prosesnya DNS server memerlukan program client yang bernama resolcer untuk menghubungkan setiap komputer user dengan server DNS. Resolver ini adalah sebuah web browser dan mail client. Berikut adalah cara kerja dari DNS server :

      1. DNS resolver akan melaksanakan pencarian alamat host pada file HOSTS, apabila alamat host yang dicari ditemukan, maka akan diberikan dan proses selesai.
      2. DNS resolver melakukan pencarian pada cache yang sebelumnya dibuat resolver sendiri untuk menampung permintaan sebelumnya, sehingga tidak lagi mengambil dari data baru.
      3. DNS resolver melakukan pencarian pada alamat server DNS yang ditentukan oleh user
      4. Server DNS berkewajiban untuk mencari nama domain pada cache yang dimilikinya.
      5. Jika nama domain yang dicari oleh server DNS ditemuka, maka pencarian dilakukan dengan melihat file database yang dimiliki oleh server.
      6. Jika tidak ditemukan maka pencarian akan dilakukan langsung dengan server yang dimaksud, dan hasilnya akan di tampilkan pada browser user.
